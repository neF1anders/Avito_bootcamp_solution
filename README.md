# Система поиска релевантных статей по запросу пользователя (Авито)

## Запуск

1. Установите зависимости:

```pip install -r requirements.txt```

2. Скачайте папку candidate_public/ с данными для задания и поместите её в корневую папку проекта.

3. Откройте `experiments.ipynb` и выполните все ячейки. Файл `answer.csv` появится в той же папке.

## Описание подхода
### Предобработка:

HTML-разметка удаляется через BeautifulSoup, текст приводится к нижнему регистру, остаются только буквы (кириллица и латиница).
Лемматизация выполняется с помощью pymorphy3, удаляются расширенный список стоп-слов (включая приветствия и частотные служебные слова) и высокочастотные термины, встречающиеся более чем в 80% статей.
Заголовок и тело объединяются, но при кодировании заголовок получает дополнительный вес (TITLE_WEIGHT=2). Вес подобран экспериментально.

### Поиск и ранжирование:

Эмбеддинги статей строятся моделью sergeyzh/rubert-mini-retriever (лёгкая русскоязычная модель для поиска). Нашел на hf, задавая фильтры на размер, дату создания и работу с русским языком.
Параллельно используется BM25 (через rank-bm25) на тексте без HTML и без лемматизации для точного совпадения терминов.
Скоры от косинусного сходства и BM25 нормализуются min‑max и комбинируются с весами W_EMB=0.55 и W_BM25=0.45 (экспериментально), после чего возвращаются топ‑10 статей.

### Калибровка:

На 500 размеченных запросах из calibration.f подбирались веса и проверялось качество (MAP). Итоговый MAP составил ~0.35.

## Что пробовал и не получилось:

Кросс-энкодер (DiTy/cross-encoder-russian-msmarco) для переранжирования кандидатов - приводил к падению MAP на 0.05, поэтому от него отказался.
Модель deepvk/USER-bge-m3 - более тяжёлая модель не дала прироста качества, но увеличила время работы.
Калибровочный сигнал (добавление веса статьям, связанным с похожими запросами из калибровки) - не принёс улучшений в экспериментах.
Агрессивное удаление стоп-слов - приводило к потере значимых слов.

Итоговое решение - гибрид эмбеддингов с BM25 без кросс-энкодера. Ответы для test.f генерируются функцией retrieve_hybrid и сохраняются в answer.csv.
